
[01] 图片支持（从视频扩到通用素材）
接下来可以考虑对图片的支持，不只是视频
单图转 ascii、批量图片转 ascii、gif/webp 这种动图也可以试

[02] 彩色 ascii（颜色模式做成可切换）
另外有对彩色 ascii 图片的支持，对彩色视频的支持
这里最好做几种模式：
1) 保留原色
2) 统一主题色（比如绿色终端风）
3) 限定调色板（少色，复古感更强）

[03] 清晰度提升（先传统增强，后模型）
超分辨率的采样，让图片/视频更加清晰
不过这个可以后放，先把传统图像增强做扎实：
亮度/对比度/锐化/边缘增强/降噪
很多时候这些就够用了，不一定一上来就上模型

[04] 输入输出适配（fps/分辨率/音轨）
不同帧率的识别，不同分辨率的变换
输入自动识别 fps + 分辨率 + 时长 + 是否有音轨
输出可以选：跟随原视频 / 强制指定 fps / 手机和网页预设分辨率

[05] 体积与编码控制（高优先级）
数据格式的问题要优先解决
现在占用空间太大，输出不能数十倍大于原视频
需要可控的编码参数（比如 crf、bitrate、preset）
并且尽量减少中间超大文件

[06] 随机风格（默认稳定，可选发散）
ascii 更加随机这个方向可以做
但最好是“可选风格模式”，默认模式还是要稳定、可读
可以做 seed，这样随机效果可复现
还可以做几种风格：matrix / glitch / noise

[07] 运行速度（先优化流程，再上 GPU）
运行速度，考虑 gpu 加速
但先做 CPU 侧优化和流程优化：
1) 关键路径 profiling（先找到慢在哪）
2) ffmpeg 管道化，减少磁盘来回读写
3) 并行处理（按帧分块/按时间段分块）
真的到瓶颈再上 cuda/opencl

[08] 预览能力（先看效果再全量渲染）
还缺几个我觉得很关键的点：

先看 1 帧、先看 3 秒、先看低清预览，再决定整段导出
不然每次全量渲染调参，时间太长

[09] 预设系统（参数可复现）
把参数保存成 preset（json/yaml）
下次直接复用，结果也更容易复现

[10] 目标模式拆分（faithful vs stylized）
faithful：尽量还原原视频内容
stylized：偏风格特效、偏艺术化
这两个目标混在一起会互相打架

[11] 一键工作流（剪辑+转换+音频）
剪辑 + ascii 转换 + 音频处理最好能一条命令串起来
以后也方便做批量处理

[12] 技术路线补充（先后顺序）
涉及到的技术可能会有：
cnn（后期再上）、GPU 加速、数据量化、调色板压缩、时域稳定、边缘感知映射

简单说，先把“体验和导出链路”做稳，再上模型和硬件加速。
